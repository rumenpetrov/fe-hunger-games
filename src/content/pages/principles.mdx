title: 'Principles'
---
* Provide 100% valid HTML
* Write [semantic HTML/CSS/JS](https://developer.mozilla.org/en-US/docs/Glossary/Semantics)
* Prefer native elements, properties or APIs before external libraries
* Use [meaningful class names](https://dev.to/stakedesigner/best-practices-for-naming-classes-in-html-and-css-files-2kb5)
* The [performance](https://web.dev/articles/vitals?hl=en) is important
* Think about the needs of the person which will use your code, the user experience(UX) and the developer experience(DX)
* Use elements, properties or APIs with sufficient browser support, required by the task you are working on
* Follow the general [best practices](https://www.linkedin.com/pulse/frontend-development-best-practices-code-structure-adekola-olawale) in web development
